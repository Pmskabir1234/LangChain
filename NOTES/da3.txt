## Structured Output in LangChain

**Structured output** in  means forcing an LLM to return data in a predictable schema instead of free-form text.

Instead of:

```text
"The user is Kabir and age is 20"
```

you get:

```json
{
  "name": "Kabir",
  "age": 20
}
```

This is useful because applications need machine-readable responses for APIs, databases, automation, agents, and workflows.

---

# Why Structured Output Matters

LLMs naturally generate unstructured text.
That creates problems:

* inconsistent formatting
* parsing errors
* hallucinated fields
* unreliable automation

Structured output solves this by enforcing schemas.

---

# Common Ways to Create Structured Output in LangChain

## 1. Using `TypedDict`

`TypedDict` is a lightweight Python typing feature.

### Example

```python
from typing_extensions import TypedDict
from langchain.chat_models import init_chat_model

class User(TypedDict):
    name: str
    age: int

llm = init_chat_model("gpt-4o-mini")

structured_llm = llm.with_structured_output(User)

result = structured_llm.invoke("Kabir is 20 years old")

print(result)
```

### Output

```python
{'name': 'Kabir', 'age': 20}
```

---

## Short Note on `TypedDict`

`TypedDict` defines dictionary structure with fixed keys and value types.

### Features

* lightweight
* fast
* simple schema definition
* no runtime validation

### Best For

* small projects
* simple schemas
* fast prototyping

### Limitation

It checks types mainly through static typing tools, not strict runtime validation.

---

# 2. Using `Pydantic`

[Pydantic](https://docs.pydantic.dev/latest/?utm_source=chatgpt.com) is more powerful and widely used for structured outputs.

It provides:

* validation
* serialization
* default values
* nested schemas
* strict type checking

---

## Example

```python
from pydantic import BaseModel
from langchain.chat_models import init_chat_model

class User(BaseModel):
    name: str
    age: int

llm = init_chat_model("gpt-4o-mini")

structured_llm = llm.with_structured_output(User)

result = structured_llm.invoke("Kabir is 20 years old")

print(result)
```

### Output

```python
User(name='Kabir', age=20)
```

---

## Short Note on Pydantic

Pydantic is a Python data validation library using type hints.

### Features

* runtime validation
* error handling
* nested objects
* optional fields
* field constraints

### Best For

* production applications
* APIs
* agents
* complex workflows

### Example Constraint

```python
from pydantic import BaseModel, Field

class Product(BaseModel):
    price: float = Field(gt=0)
```

This ensures price must be greater than 0.

# 3. Using JSON Schema

LangChain can also enforce raw JSON formats.

---

## Example

```python
schema = {
    "title": "User",
    "type": "object",
    "properties": {
        "name": {"type": "string"},
        "age": {"type": "integer"}
    },
    "required": ["name", "age"]
}
```

---

## Short Note on JSON Schema

JSON Schema is a standard format for defining JSON structures.

### Features

* language-independent
* widely supported
* API-friendly
* interoperable

### Best For

* REST APIs
* frontend-backend communication
* external integrations

---

# Comparison

| Feature            | TypedDict   | Pydantic           | JSON Schema       |
| ------------------ | ----------- | ------------------ | ----------------- |
| Runtime Validation | No          | Yes                | Depends           |
| Simplicity         | High        | Medium             | Medium            |
| Strictness         | Low         | High               | High              |
| Best Use           | Simple apps | Production systems | APIs/integrations |
| Nested Models      | Limited     | Excellent          | Excellent         |

# Main Use Cases of Structured Output

## 1. AI Agents

Agents need predictable outputs:

```json
{
  "tool": "search",
  "query": "weather Mumbai"
}
```

---

## 2. API Generation

Convert user prompts into API-ready JSON.

Example:

```json
{
  "endpoint": "/users",
  "method": "POST"
}
```

---

## 3. Database Storage

Store extracted structured data directly.

Example:

```json
{
  "customer_name": "Kabir",
  "email": "kabir@gmail.com"
}
```

---

## 4. Information Extraction

Extract entities from text.

Example:

Input:

> "Elon Musk founded SpaceX"

Output:

```json
{
  "person": "Elon Musk",
  "company": "SpaceX"
}
```

---

## 5. Workflow Automation

Used in:

* LangGraph
* AI pipelines
* tool calling
* RAG systems

---

# Key Insight

Without structured output, LLM applications become fragile because developers must parse unpredictable text.

Structured output turns LLMs into reliable data-producing systems rather than just chat generators.
